# Final project: Grocery store self-checkout

# Overview

You are tasked with creating a new touchscreen interface for grocery store self-checkouts. This project will require you to develop a clear problem statement, create an interactive prototype of the interface, and present your design solution in class. Your work aims to enhance the checkout experience by addressing common user challenges through effective UI and UX design principles.

# Assumptions for Design Context

  • You're designing for a large grocery chain planning to install landscape-oriented tablets.
  • Tablets will run your app exclusively, maintaining compatibility with existing hardware like scanners and weight-sensitive surfaces.

# Requirements

Your self-checkout app must include the following functionality:

  • Initial screens, which support the following functionality:
    • Welcome/start screen, which customers will see when they approach the checkout
    • Add your own grocery bags
    • Change language (French/English)
    • Any alert messages required by store
    • Start checking out items
  • Scanning/entering items screens, which support the following functionality:
    • Scan bar codes
    • Enter products without barcode (produce/bakery items) using code or search/browse
    • See list of products added
    • Remove item
    • Call attendant
    • Handling of errors/alerts/messages (ex. put item in bag, invalid code, etc.)
  • Complete transaction screens, which support the following functionality:
    • Indicate number of bags used
    • Scan points card
    • Select payment method
    • Error handling
    • Confirmation
  • The screen(s) supporting the specific problem you're trying to improve
    • Show where in the checkout the problem occurs
    • Show the screen(s)/elements on screen/voice instructions/etc that will improve the problem you observed

# Design Considerations

  • Reflect on the optimal way to guide users from the welcome screen to completing their transaction.
  • Ensure the interface provides clear instructions, feedback, and error handling.
  • Consider user experience for optional features and paths within the app.
  • Evaluate the necessity of navigation controls like "back" or "exit" on various screens.

# Project Components

# 1. Problem Statement Due: Wednesday April 3rd (start of class)

  • Objective: Identify a major usability issue with current self-checkout systems through a field study.
  • Format: Submit a text document outlining the observed challenge and your targeted improvement area.
  • Submission: Instructor approval required in Week 13; include in final presentation/document.
Problem statement (5x2 points)

5x2 points You will write a problem statement (text file), which will guide the rest of your work.

These must be submitted to instructor for approval in Week 13 class, and included in your final presentation/document.

Field Study: You will observe customers checking out the existing systems, and find one major difficulty that is being experienced by users. This difficulty will map to a Usability Heuristic. You will improve the checkout process by providing a solution to this difficulty.

  • Field Study Checklist document (opens new window)

Through your observed during your grocery store research trip, captured your observation in the following format:

I observed/talked to **___**person/group**___** while checking out groceries.

I noticed that **___**struggle/challenge**___**.

I want to help with **___**thing you want to improve**___**.

# 2. Interactive Prototype Due: Wednesday April 17th (start of class)

  • Objective: Design a user-friendly interface prototype addressing the identified problem.
  • Tools: Use Figma to create your prototype.
  • Key Features to Include:
    • Initial screens for welcome, language selection, alerts, and starting checkout.
    • Scanning and entering items, including error handling and attendant calls.
    • Completing transactions, selecting payment methods, and final confirmation.
  • Specific Improvement Screen(s): Demonstrate your solution to the observed problem.
  • Evaluation Criteria: Design quality, inclusion of required features, and problem-solving effectiveness.

# 3. Presentation and Prototype Walkthrough Due: Wednesday April 17th (start of class)

  • Objective: Present your design solution and prototype to the class.
  • Format: 5-minute in-class presentation with slides and a prototype demonstration.
  • Feedback: Peers provide constructive comments on your presentation and design approach.
